# Database (Supabase / Postgres)

Schema + seeds for NepaliDirectory. Apply these **in order** — later files depend on earlier ones
(the `listings` table must exist before the `ALTER TABLE listings` migrations; extensions before
`vector` columns; tables before the RLS migration).

| 3 | `migrations/V2__embeddings.sql` | `listing_embeddings` (pgvector), `category_synonyms` |

| 8 | `seeds/osm_tag_map.sql` | 150 OSM tag → category rows (generated) |

> is 1024), change the dimension in `V2` and set `EMBEDDING_DIM` to match before applying.

## How to apply

**Supabase SQL Editor** — paste each file's contents in the order above and run.

**psql** (needs `DATABASE_URL` from Supabase → Settings → Database):

```bash
for f in db/migrations/V0__listings.sql db/migrations/V1__ai_core.sql \
         db/migrations/V2__embeddings.sql db/migrations/V3__listing_enrichment.sql \
         db/migrations/V4__seed_prompt_templates.sql db/migrations/V5__acquisition.sql \
         db/migrations/V6__seed_prompts_phase2.sql db/seeds/osm_tag_map.sql \
         db/migrations/V7__rls.sql; do
  psql "$DATABASE_URL" -v ON_ERROR_STOP=1 -f "$f"
done
```

**Supabase MCP** — apply each file via the MCP's `apply_migration` / `execute_sql` tool in order.

## Regenerating the OSM seed

`seeds/osm_tag_map.sql` is generated from `lib/acquire/osm/tag-map.ts`:

```bash
node scripts/gen-osm-seed.ts
```

## Notes

- RLS: the publishable/anon key is public. Content tables (`listings` where `active=true`,
  `listing_faqs`, `category_synonyms`) allow anon `SELECT`; everything else is service-role only.
  **Any migration that CREATEs a new table must ALSO `ENABLE ROW LEVEL SECURITY` in that same
  migration** — new tables default to RLS off, and relying on an earlier blanket RLS migration
  (like V7) to cover tables created later is how V8's 4 tables briefly shipped without RLS.
- Seeding data: the 11 demo businesses from `lib/data.ts` are migrated into `listings` via
  `seeds/listings_demo.sql` (generated by `scripts/gen-listings-seed.mts`).
